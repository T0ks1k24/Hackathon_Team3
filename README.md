## 2. Встановлення залежностей

Щоб запустити проект, можна використати Docker — це значно спрощує запуск.

### Запуск через Docker

Перейдіть у папку з Docker-конфігурацією:

```bash
cd devops/docker/
docker-compose up -d
```

🔹 Backend (Python / Django)
Переконайтесь, що у вас встановлено Python 3.10+ та pip.

```bash
cd backend
python -m venv env

# Для Windows
source env/Scripts/activate

# Для MacOS/Linux
source env/bin/activate

pip install -r requirements.txt
```
🔹 Frontend (React)
Переконайтесь, що встановлено Node.js та npm.
```bash
cd frontend
npm install
```

🚀 3. Запуск проекту
🔹 Backend (Django)
```bash
cd backend
python manage.py makemigrations
python manage.py migrate
python manage.py runserver
```

🔹 Frontend (React)
```bash
cd frontend
npm start
```

Frontend буде доступний за адресою: http://localhost:3000
Backend API — на http://localhost:8000

## 📡 API

У проекті реалізовано REST API для взаємодії між фронтендом та бекендом. Усі запити здійснюються через `/api/`.

---

---

### 📚 Книги

`api/books/book/` GET - Виведеня всіх книг
`api/books/book/` POST - Створення книги
`api/books/book/{id}/` GET - Виведення книги по id
`api/books/book/{id}/` PUT - Оновлення всієї книги по id
`api/books/book/{id}/` PATCH - Оновлення книги по id
`api/books/book/{id}/` DELETE - Видалення книги
`api/books/export/` GET - Експорт даних в Excel файл
`api/books/genres/` GET - Виведення жанрів
`api/books/genres/` POST - Створення жанрів
`api/books/import-book/` POST - Виконує парсінг книг з сайту і завантажує в базу даних

---

### 👤 Користувач

`api/users/` GET - Виведення всіх користувачів
`api/users/` POST - Створення користувача
`api/users/login/` POST - Авторизація
`api/users/profile/` GET - Отримати дані профіля користувача
`api/users/reset-password/` POST - Скидання паролю користувача
`api/users/reset-password-confirm/{uidb64}/{token}/` POST - Підтвердження скидання паролю користувача
`api/users/token/refresh/` POST - Оновлення токена
`api/users/{id}/` GET - Отримання користувача по id
`api/users/{id}/` PUT - Оновлення користувача по id
`api/users/{id}/` DELETE - Видалення користувача по id

---

### 🛠️ Технічна документація

- Swagger/OpenAPI доступна за адресою:  
  [http://localhost:8000/swagger/](http://localhost:8000/swagger/)

`Authorization: Bearer <your_access_token>`
